La réalisation d'une API REST

Qu'est ce qu'une API REST ?

Le terme API d'abord signifie Application Programming Interface ou Inter de programmation en français, qui désigne un ensemble normalisé de classes, de méthodes, de fonctions et de constante qui sert de façade par laquelle un logiciel offre des services à d'autres logiciels. Ensuite, le terme REST signifie une architecture particulière d'API, "Representational State Transfer" qui désigne le fait d'utiliser une API afin de transmettre des données. Une base de données notamment peut être fourni via une API en utilisant cette architecture.

Quelle technologie utiliser pour réaliser une API REST ?

Pour réaliser un API REST on utilise la technologie NodeJS qui est une plateforme logicielle libre basée sur le language JavaScript qui a pour avantage de permettre de réaliser le backend d'une application avec ce même language.

Pourquoi s'initier à la création d'API REST ?

À ce jour, la quasi totalité des entreprises utilisent ce type de technologie pour réaliser leurs applications et la technologie NodeJS est pour cette exercice l'outil le plus puissant, le plus démocratisé et le plus stable à ce jour. Ainsi, afin de se professionnaliser, se familiariser avec cet outil semble indispensable. De cette façon, le fait de créer une API REST semble à mon sens une très bonne méthode d'apprentissage et de familiarisation d'un outil.

Mon API REST

Pour réaliser mon API REST j'ai suivi une formation vidéo de Simon Dieny disponible ici

Simon Dieny est un formateur d'ingénieurs de métier sur les technologies JavaScript et Angular chez Atos, une des plus grandes société d'informatique dans le monde

Dans cette formation vidéo d'un peu plus de 8 heures, Simon Dieny nous apprend à créer une API REST qui aura pour objectif de fournir des informations sur les pokemons au client, en utilisant ce context on apprend à :

  • Installer les outils nécessaires comme le gestionnaire de paquet NPM, l'environnement NodeJS et des dépendances
  • Comprendre et lire les réponses JSON et leur syntaxe
  • Comprendre le rôle des middlewares, notamment en utilisant Express
  • Installer phpmyadmin et le configurer pour héberger la base de données
  • Lier l'API à une base de donnée SQL (CRUD) depuis l'api en utilisant Insomnia
  • Gérer le traitement des codes d'erreurs 400 et 500
  • L'authentification du client avec JWT
  • Déployer l'API Rest en production
  • Réaliser la partie front-end avec Angular, React ou Vue
